# 母版使用指南（怎么填 template.html）

> 这个 skill 只有一个母版：`template.html`。**所有报告都用它，只换内容、不动样式。**
> 这份指南教你：① 模板有哪些"页型"、② 怎么挑页组装一份报告、③ 每个 `{{占位符}}` 填什么、④ 填写铁律。
> 配套：排版疏密 / 溢出问题看 [`layout-pitfalls.md`](layout-pitfalls.md)；完整成品看 [`../examples/mdf-acoustic-brochure/`](../examples/mdf-acoustic-brochure/)。

---

## 一、模板里有 8 种"页型"（page block），按内容挑用

每个 `<section class="page">` = 一张 A4/Letter 纸。从下面挑需要的，删掉不用的，顺序可调。

| 页型 | 长什么样 | 什么时候用 |
|---|---|---|
| **封面 cover** | 满幅大图 + 深蓝遮罩 + 大标题 + 4 个 meta chip | 必有，第 1 页 |
| **数据看板页** | 4 个大数字卡（末卡可带增长柱）+ 2×2 文字盒 + 收尾横幅 | 摘要 / 关键指标 / executive summary |
| **图片墙页** | 2×2 文字盒 + 4 图横排场景墙 + 递进行（path） | 介绍品类 / 场景 / 趋势演进 |
| **表格页** | 2×2 项目符号盒 + 5 色卡 + 三列表格 + 横幅 | 产品解剖 / 选品清单 / 对照表 |
| **产品卡页** | 2–3 张「左图右 8 格规格」强卡片 | Top N 产品 / 案例（每页最多 3 张） |
| **产品卡+洞察页** | 2 张产品卡 + 3 个橙顶边洞察盒 + 横幅（价格带图可选，默认关） | 产品卡 + 市场规律小结 |
| **步骤页** | 2×2 项目符号盒 + 编号步骤 + 横幅 | 行动建议 / 销售话术 / takeaway |
| **方法论页** | 双栏（含/不含 ▸✕）+ 来源编号 + 免责声明 | 数据口径 / 来源 / disclaimer，末页 |

> 想不重样：相邻两页别用同一种页型（别连堆 4 页 2×2 盒子）。封面=图、摘要=数字、市场=图片、产品=卡片、方法论=双栏，这样节奏才好看。

## 二、组装一份报告（推荐骨架）

**固定骨架**（几乎每份都有）：`封面` → …中间按内容选页… → `方法论`（可选）。

参考 MDF 范例的 8 页骨架：
1. 封面 → 2. 数据看板（摘要）→ 3. 图片墙（品类介绍）→ 4. 表格（产品解剖）→ 5. 产品卡 ×3 → 6. 产品卡 ×2 + 洞察盒 → 7. 步骤（takeaway）→ 8. 方法论。

**给不同读者换中间几页即可**（骨架不动）：
- 第一次进品类的新客户 → 保留"图片墙/品类介绍"做扫盲。
- 老客户季度 update → 把扫盲页换成"本季新趋势/热销更新"，重点放产品卡。

## 三、占位符地图（每个 `{{TOKEN}}` 填什么）

> 规则：填用户给的真实内容；没有的就**问用户要**，绝不自己编。

### 每个内容页都有的 5 个（每页都要填）
- `{{SECTION_TAG}}` 左上橙色栏目标签（如 EXECUTIVE SUMMARY）
- `{{SECTION_NOTE}}` 右上灰色说明（一句话点明这页讲啥）
- `{{PAGE_TITLE}}` 页大标题
- `{{FOOT_LABEL}}` 页脚左侧栏目名
- `{{PAGE_NO}}` 页脚右侧页码（02、03… 拆/合页后记得顺延）

### 封面
- `{{DOC_TITLE}}` 浏览器标题；`{{HERO_IMAGE}}`/`{{HERO_ALT}}` 封面大图（放 `images/scenes/`）
- `{{BRAND_SUBLINE}}` 品牌副行（如 Market Intelligence）
- `{{COVER_META_TOP}}`/`{{COVER_META_SUB}}` 右上角（市场 / 日期）
- `{{COVER_KICKER}}` 橙色小标签；`{{COVER_TITLE_L1}}`/`{{COVER_TITLE_L2}}` 大标题两行；`{{COVER_SUBTITLE}}` 副标题
- `{{CHIP_1_L}}`/`{{CHIP_1_V}}` … `{{CHIP_4_*}}` 四个信息块（标签/值）
- `{{COVER_FOOT_LEFT}}`/`{{COVER_FOOT_RIGHT}}` 封面底部两行

### 数据看板页
- `{{LEAD_PARAGRAPH}}` 开篇引子（可加 `<strong>` 高亮）
- `{{STAT_1_L}}`/`{{STAT_1_N}}`/`{{STAT_1_D}}` … `{{STAT_4_*}}` 四个大数字卡（标签/数字/说明）
- 末卡的增长柱（可选）：`{{BAR_1_VAL}}`/`{{BAR_1_LBL}}`、`{{BAR_2_VAL}}`/`{{BAR_2_LBL}}`（如 $9.1B / 2026 → $12.6B / 2034）
- `{{BOX_1_H}}`/`{{BOX_1_P}}` … `{{BOX_4_*}}` 2×2 文字盒（标题/正文）
- `{{BANNER_TITLE}}`/`{{BANNER_BODY}}` 深蓝收尾横幅

### 图片墙页
- `{{BOX_1_*}}`…`{{BOX_4_*}}` 2×2 盒
- `{{GALLERY_SUBHEAD}}` 小标题；`{{G1_IMG}}`/`{{G1_ALT}}`/`{{G1_H}}`/`{{G1_P}}` … `{{G4_*}}` 四张场景图（图/alt/标题/一行描述）
- `{{PATH_SUBHEAD}}`；`{{PATH_1_STEP}}`/`{{PATH_1_TXT}}` … `{{PATH_3_*}}` 三条递进行

### 表格页
- `{{BOX_1_H}}` + `{{BOX_1_LI_1..3}}` … `{{BOX_4_*}}` 2×2 项目符号盒
- `{{SWATCH_SUBHEAD}}`；`{{SW1_HEX}}`/`{{SW1_NM}}` … `{{SW5_*}}` 五个色卡（**填真实色号**，如木纹色）
- `{{TABLE_SUBHEAD}}`；表头 `{{TH_1..3}}`；行 `{{R1_C1..C3}}` … `{{R5_*}}`（可增删 `<tr>`）
- `{{BANNER_TITLE}}`/`{{BANNER_BODY}}`

### 产品卡（每张卡一组，可复制 `<article>`）
- `{{RANK}}` 排名数字；`{{P_IMG}}`/`{{P_ALT}}` 产品图（放 `images/products/`）
- `{{P_TITLE}}` 产品标题；`{{P_MARKET}}` 市场药丸（如 Market: US）；`{{P_BRAND}}` 品牌
- `{{SPEC_1_L}}`/`{{SPEC_1_V}}` … `{{SPEC_8_*}}` 8 格规格（标签/值；第 1 格自带橙色价格样式、第 8 格自带等宽 ASIN 样式）
- `{{P_NOTE}}` 卡内说明；`{{P_SOURCE}}` 来源行

### 产品卡+洞察页（独有）
- 产品卡同上
- 价格带图（**可选，默认注释掉**；加上会变挤，要加先看 pitfalls 坑 5）：`{{PRICEBAND_SUBHEAD}}`、`{{PB_TICK_1..4}}`（刻度）、`{{PB_ZONE_1}}`/`{{PB_ZONE_2}}`（区间标签）
- `{{PATTERN_SUBHEAD}}` 洞察盒小标题；`{{PAT_1_H}}`/`{{PAT_1_P}}` … `{{PAT_3_*}}` 三个橙顶边洞察盒
- `{{BANNER_*}}`

### 步骤页
- `{{BOX_1_H}}` + `{{BOX_1_LI_1..4}}` … `{{BOX_4_*}}`
- `{{STEPS_SUBHEAD}}`；`{{STEP_1_K}}`/`{{STEP_1_T}}` … `{{STEP_4_*}}`（编号标签/正文）
- `{{BANNER_*}}`

### 方法论页
- `{{METHOD_L_TITLE}}` + `{{METHOD_L_1..4}}`；`{{METHOD_R_TITLE}}` + `{{METHOD_R_1..4}}`（某条加 `class="ex"` 显示 ✕）
- `{{SOURCES_SUBHEAD}}`；`{{SRC_1_B}}`(加粗来源名)/`{{SRC_1_T}}`(说明) … `{{SRC_6_*}}`（可增删）
- `{{DISCLAIMER_TEXT}}` 免责声明

## 四、填写铁律（4 条，违反就翻车）

1. **只填真实内容**：用户给什么填什么；缺就问，绝不编数据/文案/结论。
2. **图表只可视化"已有数字"**：增长柱 / 价格带 / 色卡用的数值颜色，必须是原文已写明的；原文没有就别画。
3. **图片放对目录**：`images/scenes/`（封面+场景）、`images/products/`（产品图）；用相对路径，渲染时用 `file://绝对路径/index.html`。
4. **改完逐页读 PDF 验证**：固定页会裁内容且不报错——见 [`layout-pitfalls.md`](layout-pitfalls.md) §4 清单。

---

## 五、绝不能动的（UI 锁死）

主色 / 字体（Inter）/ 圆角 / 页面尺寸 / 各 block 的样式（卡片色条、橙顶边、横幅、规格面板…）——**一律不改**。母版长什么样，所有报告就长什么样。
唯一例外：某页内容溢出时，**仅为塞进一页**可微调该页间距（见 pitfalls §3 旋钮表）。
