# 排版避坑手册（intco-decor-report）

> 用这个母版填内容、出 PDF 时的实战经验。**套模板生成报告前先读这份**，能少走弯路、一次到位。
> 这些坑都是真实踩过的——记下来是为了后面别再踩。

## 0. 一句话铁律

报告是**固定页**（816×1056px / Letter）+ `overflow:hidden`。内容一旦超出一页，就会被**悄悄裁掉、不报任何错**。
所以：**每改一次，必须重新生成 PDF 并逐页读图确认**——不能凭 HTML 脑补、不能只看一两页。
（这一条是所有坑的根源；下面 7 个坑本质都是它的变体。）

---

## 1. 七个真实踩过的坑（症状 → 原因 → 解法）

### 坑 1 · 内容超一页被裁，还不报错【最高频】
- **症状**：某页最后一行 / 列表最后一项 / "market path"最后一行 / 横幅第二行——不见了或贴底边切掉。
- **原因**：`.page` 固定高 1056px + `overflow:hidden`，超出部分直接裁掉。
- **解法**：① 逐页读 PDF 检查；② 收紧该页间距（见 §3）；③ 还超就拆页。

### 坑 2 · 页脚和内容/横幅重叠
- **症状**：深蓝横幅压住页脚文字。
- **原因**：页脚曾用 `position:absolute`，脱离文档流。
- **解法**：页脚走**正常流式**（`.foot{ flex-shrink:0 }`，排在 `.body` 之后）——物理上不可能再和内容重叠。模板已是这样，**别改回绝对定位**。

### 坑 3 · 徽章/卡片的投影在图片上糊成色块
- **症状**：rank 徽章周围冒出一个橙色 / 灰色方块（压在深色产品图上、PDF 放大看尤其明显）。
- **原因**：`box-shadow`（无论品牌橙还是中性灰）在照片背景上经 PDF 渲染会糊成一坨。
- **解法**：**压在图片上的元素一律不用 box-shadow**。卡感靠：边框 + 左侧深蓝色条 + 带底色的规格面板。模板的 `.pcard` / `.pcard .rank` 已去阴影，**别加回去**。

### 坑 4 · 内容"1.x 页"——塞一页会挤、拆两页会空
- **症状**：硬塞一页 → 横幅顶到上面内容（挤）；拆两页 → 两页都半空。
- **原因**：内容量正好"一页多一点"，是最难的情况。
- **解法**：先判断**真实份量**。
  - ≈1 页 → **精确收进一页**：收紧间距 + 结尾横幅 `margin-top:auto` 吸底，富余空间自动变成横幅上方的呼吸位。
  - 真有 2 页的量 → 才拆。
  - **别把 1.1 页的东西硬拆成 2 页**（会得到两个半空页）。

### 坑 5 · 页面太挤时，先砍"重复的可视化"，别砍数据
- **例**：价格带图 和 "Price band is broad" 文字盒讲的是同一组数字（$50–$110 / C$100–C$360）→ 砍掉那张图，数据一点不丢，页面立刻松开。
- **解法**：密度超标时，先找"有没有图/表在重复已经有文字写出来的数字"，优先删那张图，而不是删内容。

### 坑 6 · 弱元素 + 没标题 = 飘
- **症状**：底部 3 个灰平板盒子，没标题、样式比上面的卡片弱 → 整页头重脚轻、盒子像凭空冒出来。
- **解法**：① 给它一个小标题（`.subhead`）定性（如 "Amazon Pattern for Importers"）；② 让它**套用全报告已有的视觉语言**（如橙色顶边 `border-top:3px solid var(--accent)`，和第 2 页数据卡一致）。**统一 > 各做各的**。

### 坑 7 · `margin-top:auto` 吸底横幅是双刃剑
- 内容**刚好满** → 没富余空间可分 → 横幅紧贴上面内容（挤）。
- 内容**太少** → 富余太多 → 横幅离得老远（飘）。
- **解法**：让该页内容量**接近但略小于**一页，留 ~40–80px 给横幅上方当呼吸位。

## 2. 改 CSS 前先分清：全局 class vs 单页 class

调**单页 class** 可放心；调**全局 class** 会动到所有用它的页，改完必须把那几页都重新验证。

- **只在某一页用（可放心调）**：
  - `.pat` / `.pat .b`（仅 pattern 洞察盒页）
  - `.priceband` 和价格带 `<svg height>`（仅该页，且 height 是写在 HTML 里的 inline 值）
  - `.swatches`（仅产品解剖页色卡）
  - `.method` / `.sources` / `.disc`（仅方法论页）
  - `.conv` / `.crow`（仅 takeaway 页步骤）
  - `.stats` / `.stat`（仅摘要页）
  - `.gallery` / `.gcard`（仅图片墙页）
- **全局（慎调，改完逐页重验）**：
  - `.page` `.body` `.foot` `.eyebrow` `.ptitle` `.rule` `.lead`
  - `.banner`（多页收尾都用）
  - `.box` / `.boxes`（多页 2×2 盒子都用）
  - `.subhead`（多页小标题都用）
  - `.pcard` 及其子元素（多张产品页都用）

## 3. 填满 / 收紧一页的常用旋钮（按影响从大到小）

**收紧（内容超了）**：
- 减 padding：`.box` / `.pcard .pb` / `.tbl td` / `.banner` / `.crow` / `.pat .b`
- 减 gap：`.boxes` / `.pcards`
- 减价格带 `<svg height>`
- 图片比例改矮：gallery `.gcard .im` 用 `aspect-ratio:4/3` 比 `3/4` 矮一截

**填空（内容太少）**：
- 结尾 `.banner` 用 `margin-top:auto` 吸到页底
- 轻内容用 `<div style="flex:1; display:flex; align-items:center">` 包起来垂直居中
- 适当放大盒子 padding / 字号

**硬上限**：每页最多 **2–3 张产品卡**；「2 卡 + 价格带 + 3 盒 + 横幅」就是一页的极限，再多必拆。

## 4. 每次生成 PDF 后的验证清单【必做，逐页】

- [ ] `pdfinfo <pdf> | grep Pages` 页数 = 预期，**无尾随空白页**
- [ ] 逐页读图：**没有文字被裁**（重点查每页最后一行 / 列表末项 / 横幅第二行）
- [ ] 横幅**没压页脚**、两行完整
- [ ] 卡片/徽章周围**没有灰/橙色块**（投影糊块）
- [ ] **没有半空页**（内容飘在中间、或下半页大片空白）
- [ ] 拆页/合页后，**页码 footer 顺延正确**（02、03…连续）
- [ ] 改了范例 → **同步进 `template.html`**（母版和产出必须一致）
- [ ] 图表只**可视化原文已有数字**，没编新数据

---

## 5. 一句话总结这次的教训

> 固定分页排版，"内容刚好一页多一点"时最坑——**要么精确收进一页、要么真有两页的量再拆；中间没有又松又不空的免费午餐**。唯一可靠的办法是**改完逐页读 PDF**，凭感觉一定翻车。
